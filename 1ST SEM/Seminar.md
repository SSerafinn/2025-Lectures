**Content:**
Every program ever written is built on just **4 core concepts:**

1. 🔢 **Sequence** — Order matters!
1. 🔁 **Loops** — Do it again (and again)
1. ❓ **Conditionals** — Make a decision

## SLIDE 7 — Concept 1: Sequence

**Title:** 1. Sequence 🔢

**Content:**

> **Sequence** = Instructions must be in the correct **ORDER**.

A computer follows steps **exactly as written — top to bottom.**

**Example: Getting Ready in the Morning**

1. Wake up
1. Brush teeth
1. Put on socks 🧦
1. Put on shoes 👟

> ❌ *What happens if you put on shoes BEFORE socks?*

> *“In programming, wrong order = wrong result. We call a mistake in code a BUG 🐛”*

> 🎨 *Visual: Fun illustration of someone wearing shoes over bare feet with a ❌ sign.*

-----

## SLIDE 8 — Concept 2: Loops

**Title:** 2. Loops 🔁

**Content:**

> **Loop** = Repeat a set of instructions **multiple times** automatically.

**Without a Loop:**

```
Move 1 step
Move 1 step
Move 1 step
... (100 times!) 😫
```

**With a Loop:**

```
Repeat 100:
  Move 1 step   ✅ Much easier!
```

**Real-life example:**
🌀 Washing Machine: Spin → Rinse → Drain → *repeat until clean*

> 🎨 *Visual: A circular arrow “loop” graphic. Side-by-side comparison of with/without loop.*

-----

## SLIDE 9 — Concept 3: Conditionals

**Title:** 3. Conditionals ❓

**Content:**

> **Conditional** = The computer makes a **DECISION** based on whether something is TRUE or FALSE.

**The Pattern:**

```
IF (condition is true)
  → do this
ELSE
  → do that instead
```

**Real-life example:**

```
IF it is raining 🌧️
  → Bring an umbrella ☂️
ELSE
  → Wear sunglasses 😎
```

**In games:**

```
IF score reaches 100
  → Show "YOU WIN!" 🏆
ELSE
  → Keep playing
```

> 🎨 *Visual: A flowchart diagram showing IF/ELSE branching paths.*

-----

## SLIDE 10 — Concept 4: Variables

**Title:** 4. Variables 📦

**Content:**

> **Variable** = A named **container** that stores information which can change.

Think of it like a **labeled box** 📦:

```
┌─────────────┐
│   Score     │  ← the label (variable name)
│     25      │  ← the value (can change anytime)
└─────────────┘
```

**Common uses in games:**

- 🏅 **Score** — increases when you earn points
- ⏱️ **Timer** — counts down from 30
- ❤️ **Lives** — decreases when you get hit
- 🏆 **High Score** — remembers your best run

> 🎨 *Visual: Cute labeled box illustration. Or a game HUD showing Score, Timer, Lives.*

## SLIDE 16 — Step 1: Make It Move

**Title:** Step 1 — Make Your Sprite Move 🏃

**Content:**
Find these blocks and snap them together:

```
When 🚩 clicked          ← Events (yellow)
Forever                  ← Control (orange) = LOOP!
  Move (10) steps        ← Motion (blue)
  If on edge, bounce     ← Motion (blue)
```

▶️ **Click the Green Flag and test it!**

> ✅ *Your sprite should be bouncing around the screen!*

> 🗣️ *Ask: “Which of our 4 concepts is the ‘Forever’ block?”* → Answer: Loop!

> 🎨 *Visual: Screenshot of the exact blocks in Scratch, color-highlighted.*

-----

## SLIDE 17 — Step 2: Chase the Mouse

**Title:** Step 2 — Follow the Mouse 🖱️

**Content:**
Change your code to:

```
When 🚩 clicked          ← Events (yellow)
Forever                  ← Control (orange)
  Point towards          ← Motion (blue)
    (mouse-pointer ▾)
  Move (5) steps         ← Motion (blue)
```

▶️ **Move your mouse around the stage. Does it follow?**

> ✅ *Your sprite should now chase your mouse cursor!*

> 🗣️ *Ask: “What concept makes the sprite do this over and over?”* → Answer: Loop!

> 🎨 *Visual: Screenshot of updated blocks. Arrow showing mouse → sprite direction.*

-----

## SLIDE 18 — Step 3: Add a Reaction

**Title:** Step 3 — Make It React! 💬

**Content:**
Add inside your Forever loop:

```
If <touching (mouse-pointer ▾)?>   ← Sensing + Control = CONDITIONAL!
  Play sound (Meow ▾) until done   ← Sound (red)
  Say [Got you!] for (1) seconds   ← Looks (purple)
  Wait (1) seconds                 ← Control (orange)
```

▶️ **Try to escape! What happens when it catches you?**

> ✅ *Sprite should speak and play a sound when it catches the cursor!*

> 🗣️ *Ask: “Which concept is the ‘If touching’ block?”* → Answer: Conditional!

> 🎨 *Visual: Screenshot of the If block highlighted. Speech bubble on sprite.*

-----

## SLIDE 19 — Step 4: Add a Score

**Title:** Step 4 — Keep Score! 🏅

**Content:**
**First:** Go to **Variables tab** → Click **“Make a Variable”** → Name it **Score**

Then add inside your Forever loop:

```
Change (Score) by (1)     ← Variables (dark orange)
Wait (1) seconds          ← Control (orange)
```

▶️ **Start the game. Does your score go up every second?**

> ✅ *Score should increase the longer you survive!*

> 🗣️ *Ask: “What concept is ‘Score’?”* → Answer: Variable!

## BONUS SLIDE — What’s a Bug? 🐛

**Title:** What’s a Bug? 🐛

**Content:**

> A **bug** is a mistake in code that causes a program to behave unexpectedly.

**Famous story:** The word “bug” came from 1947 when engineers found a real moth stuck inside a computer causing errors!

**Finding and fixing bugs = DEBUGGING 🔍**

> *“When your code doesn’t work, you’re not failing — you’re debugging. That’s exactly what programmers do all day.”*

> 🎨 *Visual: Photo of the original 1947 moth bug in the logbook (historical). Fun cartoon bug.*
