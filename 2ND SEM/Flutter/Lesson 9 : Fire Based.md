## What We Are Building Today

By the end of this lecture, your app will:

1. Connect to a real **Firebase** project in the cloud
2. Let a user **create an account** (sign up) with email and password
3. Let a returning user **sign in**
4. Land the user on a simple **Home screen** after authentication
5. Let the user **sign out** and return to the login screen

We are using **Firebase Authentication** with the **Email/Password** provider. No database yet — that's a future lecture. The single goal today is: *get our Flutter app talking to Firebase.*

> **Mental model:** Firebase is Google's backend-as-a-service. Instead of writing and hosting our own server + user database, we let Firebase handle account storage, password hashing, and session tokens. Our Flutter app just makes calls to it.

## Part 2 — Install the CLI Tools (one-time setup)

We connect Flutter to Firebase using two command-line tools: the **Firebase CLI** and the **FlutterFire CLI**. This is the modern, recommended approach — it auto-generates all the platform config so we don't hand-edit Gradle or plist files.

**Step 2.1 — Install the Firebase CLI** (run from any directory):

```bash
npm install -g firebase-tools
```

**Step 2.2 — Log in to Firebase** (this opens a browser to authorize):

```bash
firebase login
```

**Step 2.3 — Install the FlutterFire CLI:**

```bash
dart pub global activate flutterfire_cli
```

> **Note for Windows/macOS:** If `flutterfire` is "not recognized" after this, your Dart pub global path isn't on your system PATH. On macOS/Linux add `export PATH="$PATH":"$HOME/.pub-cache/bin"` to your shell profile. On Windows, add `%LOCALAPPDATA%\Pub\Cache\bin` to your PATH environment variable, then restart the terminal.

## Part 3 — Create the Flutter Project & Add Packages

**Step 3.1 — Create a fresh project:**

```bash
flutter create firebase_auth_demo

```

**Step 3.2 — Add the two Firebase packages we need:**

```bash
flutter pub add firebase_core firebase_auth
```

- `firebase_core` — required by *every* Firebase app; it initializes the connection.
- `firebase_auth` — the authentication features (sign up, sign in, sign out).

**Step 3.3 — Connect this project to your Firebase project:**

```bash
flutterfire configure
```

You'll be prompted to:

- **Select a Firebase project** → choose `ite124-auth-demo` (use arrow keys).
- **Select platforms** → use the spacebar to tick **android** (and **ios** / **web** if you're targeting those), then Enter.

When it finishes, it generates a file at **`lib/firebase_options.dart`**. **Do not edit this file by hand** — it holds your project's connection keys.

> **If you ever change platforms or projects later**, just re-run `flutterfire configure`.

## Part 4 — Initialize Firebase in `main.dart`

Open `lib/main.dart` and **replace its entire contents** with the starter below. This wires up Firebase on startup and decides which screen to show first.

```dart
import 'package:flutter/material.dart';
import 'package:firebase_core/firebase_core.dart';
import 'package:firebase_auth/firebase_auth.dart';
import 'firebase_options.dart';

import 'screens/auth_screen.dart';
import 'screens/home_screen.dart';

Future<void> main() async {
  // Ensure Flutter is ready before we do async work in main().
  WidgetsFlutterBinding.ensureInitialized();

  // Connect to Firebase using the generated config.
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Firebase Auth Demo',
      debugShowCheckedModeBanner: false,
      theme: ThemeData(
        colorScheme: ColorScheme.fromSeed(seedColor: Colors.indigo),
        useMaterial3: true,
      ),
      // The "gatekeeper": listen to auth state and route accordingly.
      home: const AuthGate(),
    );
  }
}

/// Decides which screen to show based on whether a user is logged in.
/// This rebuilds automatically whenever the user logs in or out.
class AuthGate extends StatelessWidget {
  const AuthGate({super.key});

  @override
  Widget build(BuildContext context) {
    return StreamBuilder<User?>(
      stream: FirebaseAuth.instance.authStateChanges(),
      builder: (context, snapshot) {
        // Still checking the auth state.
        if (snapshot.connectionState == ConnectionState.waiting) {
          return const Scaffold(
            body: Center(child: CircularProgressIndicator()),
          );
        }

        // If we have a user, go Home. Otherwise show the auth screen.
        if (snapshot.hasData) {
          return const HomeScreen();
        }
        return const AuthScreen();
      },
    );
  }
}
```

> **The key idea here is `authStateChanges()`.** It's a *stream* — Firebase pushes us an event every time the login state changes. We don't manually navigate after login; the `StreamBuilder` rebuilds and the `AuthGate` swaps the screen for us. This is the cleanest pattern for auth routing.

## Part 6 — Wire the Form to Firebase Auth

Now we replace the empty `_submit()` method. In `screens/auth_screen.dart`, **add the import at the top:**

```dart
import 'package:firebase_auth/firebase_auth.dart';
```

Then **replace the `_submit()` method** with this:

```dart
Future<void> _submit() async {
  final email = _emailController.text.trim();
  final password = _passwordController.text.trim();

  // Basic client-side check before calling Firebase.
  if (email.isEmpty || password.isEmpty) {
    setState(() => _errorMessage = 'Please fill in all fields.');
    return;
  }

  setState(() {
    _isLoading = true;
    _errorMessage = null;
  });

  try {
    if (_isLogin) {
      // Existing user signs in.
      await FirebaseAuth.instance.signInWithEmailAndPassword(
        email: email,
        password: password,
      );
    } else {
      // New user account is created.
      await FirebaseAuth.instance.createUserWithEmailAndPassword(
        email: email,
        password: password,
      );
    }
    // On success: we do NOT navigate manually.
    // The AuthGate's StreamBuilder detects the change and shows HomeScreen.
  } on FirebaseAuthException catch (e) {
    // Firebase gives us a specific error code we can show the user.
    setState(() {
      _errorMessage = e.message ?? 'Authentication failed.';
    });
  } finally {
    // Only update state if the widget is still on screen.
    if (mounted) {
      setState(() => _isLoading = false);
    }
  }
}
```

> **Two methods do all the work:**
> - `createUserWithEmailAndPassword(...)` → makes a brand new account.
> - `signInWithEmailAndPassword(...)` → logs in an existing account.
>
> Both are `async` and can throw a `FirebaseAuthException` (e.g. "email already in use", "wrong password", "password too weak"). We catch it and show `e.message` to the user. **Never let these throw uncaught** — that's a crash and a bad demo.

## Part 7 — The Home Screen

Create **`lib/screens/home_screen.dart`**. It greets the logged-in user by email and offers a sign-out button.

```dart
import 'package:flutter/material.dart';
import 'package:firebase_auth/firebase_auth.dart';

class HomeScreen extends StatelessWidget {
  const HomeScreen({super.key});

  @override
  Widget build(BuildContext context) {
    // Grab the currently logged-in user. Safe here because AuthGate
    // only shows HomeScreen when a user exists.
    final user = FirebaseAuth.instance.currentUser;

    return Scaffold(
      appBar: AppBar(
        title: const Text('Home'),
        actions: [
          IconButton(
            icon: const Icon(Icons.logout),
            tooltip: 'Sign out',
            onPressed: () async {
              // After this, AuthGate detects logout and returns to AuthScreen.
              await FirebaseAuth.instance.signOut();
            },
          ),
        ],
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            const Icon(Icons.check_circle, size: 72, color: Colors.green),
            const SizedBox(height: 16),
            const Text(
              'You are signed in!',
              style: TextStyle(fontSize: 22, fontWeight: FontWeight.bold),
            ),
            const SizedBox(height: 8),
            Text(
              user?.email ?? 'No email',
              style: const TextStyle(fontSize: 16, color: Colors.black54),
            ),
            const SizedBox(height: 4),
            Text(
              'UID: ${user?.uid ?? "unknown"}',
              style: const TextStyle(fontSize: 12, color: Colors.black38),
            ),
          ],
        ),
      ),
    );
  }
}
```

> **Notice `signOut()`** is all we call. We never push or pop a route — the `AuthGate` stream handles the navigation back to the login screen automatically. This is why the stream-based pattern is so clean.

## Common Errors & Fixes

| Symptom | Likely cause | Fix |
|---|---|---|
| `flutterfire: command not found` | Pub global bin not on PATH | See the PATH note in Part 2 |
| Crash on launch / `No Firebase App` | `Firebase.initializeApp` not awaited, or `firebase_options.dart` missing | Re-check Part 4; re-run `flutterfire configure` |
| "operation-not-allowed" on sign-up | Email/Password provider not enabled | Enable it in Console (Part 1) |
| "weak-password" | Password under 6 characters | Use a longer password |
| "email-already-in-use" | Account already exists | Switch to Log In mode |
| Build fails on Android (minSdk) | Firebase needs a higher minSdk | Set `minSdk = 23` in `android/app/build.gradle` |

## Recap

Today you learned to:

- Create a Firebase project and enable Email/Password auth
- Connect Flutter to Firebase with the FlutterFire CLI (no manual config files)
- Initialize Firebase in `main.dart`
- Use `createUserWithEmailAndPassword`, `signInWithEmailAndPassword`, and `signOut`
- Route screens reactively with `authStateChanges()` and a `StreamBuilder`

**Key takeaway:** authentication state is a *stream*. We listen to it once at the top of the app and let the UI react — we never manually navigate on login or logout.
